步骤 1：在后端项目中添加周报生成逻辑

// 在 CloudFlare Workers 中添加
async function generateWeeklyBlog() {
  // 1. 获取过去 7 天的日报内容
  const dailyReports = await fetchLast7DaysReports();
  
  // 2. 使用 AI 生成周报
  const blogContent = await generateBlogWithAI(dailyReports);
  
  // 3. 推送到前端仓库
  await pushToGitHub({
    repo: 'dongyu19920904/Hextra-AI-Insight-Daily',
    path: `content/blog/${getWeekFileName()}.md`,
    content: blogContent
  });
}

// 定时触发（每周日 22:00）
addEventListener('scheduled', event => {
  event.waitUntil(generateWeeklyBlog());
});
步骤 2：前端项目无需修改
前端项目会自动：
检测到 content/blog/ 目录有新文件
GitHub Actions 自动构建
博客自动显示在网站上
💡 为什么要在后端实现？
原因 1：统一的内容生成逻辑
后端已经有每日资讯生成的完整流程
周报只是对每日内容的汇总
复用现有的 AI 调用和 GitHub 推送逻辑
原因 2：定时任务管理
CloudFlare Workers 支持 Cron 定时任务
前端 GitHub Actions 也可以，但后端更灵活
原因 3：数据访问
后端可以直接访问过去 7 天的日报数据
前端需要额外的 API 调用
